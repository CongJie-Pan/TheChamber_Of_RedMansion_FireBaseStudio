# AI 題目生成 Prompts

本文件提供用於生成紅樓夢每日修身題庫的 AI Prompts。
請將生成的題目填入 `question-bank.json` 對應的類型和難度中。

---

## 📋 通用說明

### 難度定義
- **easy (簡單)**: 適合初學者，直接從文本找答案，不需要深度分析
- **medium (中等)**: 需要一定理解力，涉及人物關係或情節連結
- **hard (困難)**: 需要深度分析、跨章節理解、或文學批評能力

### 輸出格式要求
- 所有輸出必須是 **有效的 JSON 格式**
- 使用**繁體中文**
- 每個難度建議生成 **10-15 題**

## 1️⃣ 晨讀時光 (morning_reading) - TextPassage

### Prompt

```
你是一位紅樓夢專家，請為「晨讀時光」任務生成閱讀理解題目。

## 任務說明
用戶會閱讀一段《紅樓夢》原文，然後回答理解性問題。系統會用 AI 評分用戶的回答。

## 輸出格式 (JSON Array)
請生成 [DIFFICULTY] 難度的題目，每題格式如下：

```json
{
  "chapter": 章回編號(數字),
  "startLine": 起始行號(數字),
  "endLine": 結束行號(數字),
  "text": "原文內容（150-300字為佳）",
  "question": "閱讀理解問題？",
  "hint": "思考提示：引導學生思考的方向...",
  "expectedKeywords": ["關鍵詞1", "關鍵詞2", "關鍵詞3", "關鍵詞4", "關鍵詞5"]
}
```

## 難度要求

### Easy (簡單)
- 選擇第 1-10 回的經典段落
- 問題直接從文本可找到答案
- 例如：「這段文字描述了什麼場景？」「黛玉在這段中表現出什麼情緒？」

### Medium (中等)
- 選擇第 1-40 回的段落
- 問題需要理解人物心理或情節意義
- 例如：「作者為何這樣描寫？」「這段話暗示了什麼？」

### Hard (困難)
- 可選擇任意章回
- 問題需要深度分析、文學鑑賞、或跨章節理解
- 例如：「這段如何與全書主題呼應？」「從文學手法角度分析這段描寫」

## 生成要求
1. 請生成 10 題 [DIFFICULTY] 難度的題目
2. 題目涵蓋不同章回，避免重複
3. 原文選擇具有代表性的經典段落
4. 問題設計要有教育意義
5. expectedKeywords 提供 3-5 個關鍵詞，用於 AI 評分參考

請直接輸出 JSON Array，不要加其他說明文字。
```

### 範例輸出

```json
[
  {
    "chapter": 3,
    "startLine": 1,
    "endLine": 10,
    "text": "卻說黛玉自那日棄舟登岸時，便有榮國府打發了轎子並拉行李的車輛久候了。這林黛玉常聽得母親說過，他外祖母家與別家不同。他近日所見的這幾個三等僕婦，吃穿用度，已是不凡了，何況今至其家。因此步步留心，時時在意，不肯輕易多說一句話，多行一步路，惟恐被人恥笑了他去。",
    "question": "從這段文字中，你認為林黛玉初到賈府時的心理狀態是什麼？她為什麼會有這樣的心理？",
    "hint": "思考提示：注意黛玉的行為描述「步步留心，時時在意」，以及她擔心「被人恥笑」的心理。想想看，她為什麼會如此小心翼翼？這反映了她什麼樣的處境？",
    "expectedKeywords": ["謹慎", "小心", "拘謹", "寄人籬下", "步步留心"]
  }
]
```

## 2️⃣ 人物洞察 (character_insight) - CharacterPrompt

### Prompt

```
你是一位紅樓夢專家，請為「人物洞察」任務生成角色分析題目。

## 任務說明
用戶需要分析《紅樓夢》中的人物性格、關係、命運等，提交分析文章由 AI 評分。

## 輸出格式 (JSON Array)
請生成 [DIFFICULTY] 難度的題目，每題格式如下：

```json
{
  "characterId": "角色ID(英文小寫)",
  "characterName": "角色中文名",
  "analysisPrompts": [
    "分析問題1？",
    "分析問題2？",
    "分析問題3？"
  ],
  "chapter": 相關章回(數字，可選),
  "context": "情境背景描述"
}
```

## 主要角色 ID 對照表
- daiyu: 林黛玉
- baoyu: 賈寶玉
- baochai: 薛寶釵
- xifeng: 王熙鳳
- xiangyun: 史湘雲
- tanchun: 賈探春
- xichun: 賈惜春
- yingchun: 賈迎春
- miaoyu: 妙玉
- qiaojie: 巧姐
- jia_mu: 賈母
- wang_furen: 王夫人
- xue_yi_ma: 薛姨媽
- liu_laolao: 劉姥姥
- qingwen: 晴雯
- xiren: 襲人
- zijuan: 紫鵑
- ping_er: 平兒
- jia_zheng: 賈政
- jia_lian: 賈璉

## 難度要求

### Easy (簡單)
- 選擇主要角色（寶玉、黛玉、寶釵、鳳姐）
- 問題聚焦單一性格特點或具體事件
- 1-2 個分析問題
- 例如：「林黛玉的敏感性格如何體現？」

### Medium (中等)
- 可選擇次要角色
- 問題涉及人物關係或性格多面性
- 2-3 個分析問題
- 例如：「分析王熙鳳的管理才能與為人處世」

### Hard (困難)
- 可選擇任何角色
- 問題涉及命運分析、跨角色比較、或與主題的關聯
- 3-4 個分析問題
- 例如：「比較黛玉與寶釵的處世哲學及其命運結局」

## 生成要求
1. 請生成 10 題 [DIFFICULTY] 難度的題目
2. 涵蓋不同角色，避免重複
3. context 提供具體的故事情境
4. analysisPrompts 的問題要有層次感，引導深入思考

請直接輸出 JSON Array，不要加其他說明文字。
```

### 範例輸出

```json
[
  {
    "characterId": "daiyu",
    "characterName": "林黛玉",
    "analysisPrompts": [
      "林黛玉初入賈府時的性格特點是什麼？",
      "她的「小心眼」是任性還是自我保護？"
    ],
    "chapter": 3,
    "context": "林黛玉初入賈府"
  },
  {
    "characterId": "xifeng",
    "characterName": "王熙鳳",
    "analysisPrompts": [
      "王熙鳳的管理才能與權謀手段如何體現？",
      "她的悲劇命運是性格使然還是時代必然？",
      "分析「機關算盡太聰明，反算了卿卿性命」的深層含義。",
      "她與賈璉的婚姻關係折射出怎樣的社會現實？"
    ],
    "chapter": 13,
    "context": "協理寧國府"
  }
]
```

## 3️⃣ 文化探秘 (cultural_exploration) - CulturalElement

### Prompt

```
你是一位紅樓夢專家，請為「文化探秘」任務生成文化知識測驗題目。

## 任務說明
用戶學習《紅樓夢》中的傳統文化元素，包括禮儀、服飾、建築、飲食、詩詞、戲曲等，然後回答測驗問題。

## 輸出格式 (JSON Array)
請生成 [DIFFICULTY] 難度的題目，每題格式如下：

```json
{
  "id": "culture_XXX",
  "category": "類別",
  "title": "文化元素標題",
  "description": "文化元素的詳細描述（200-400字）",
  "relatedChapters": [相關章回1, 相關章回2],
  "questions": [
    {
      "id": "q1",
      "question": "測驗問題？",
      "type": "multiple_choice",
      "options": ["選項A", "選項B", "選項C", "選項D"],
      "correctAnswer": "正確選項",
      "explanation": "答案解釋"
    }
  ]
}
```

## 文化類別
- 禮儀：見面禮節、稱謂、拜訪禮儀等
- 服飾：衣著、配飾、妝容等
- 建築：園林、房屋、裝飾等
- 飲食：菜餚、茶酒、飲食禮儀等
- 詩詞：詩社、詩作、文學典故等
- 戲曲：戲文、曲詞、表演等
- 節慶：節日習俗、慶典活動等
- 器物：文房四寶、古玩、家具等
- 醫藥：中醫、藥方、養生等
- 宗教：佛教、道教、民間信仰等

## 題型說明
- multiple_choice: 單選題，提供 4 個選項
- true_false: 是非題，correctAnswer 為 "true" 或 "false"
- short_answer: 簡答題，correctAnswer 可以是字串陣列（多個可接受答案）

## 難度要求

### Easy (簡單)
- 基礎文化知識
- 1 個 multiple_choice 題目
- 問題直接從 description 可找到答案

### Medium (中等)
- 進階文化知識
- 1-2 個題目，可混合題型
- 需要理解文化背景

### Hard (困難)
- 深度文化知識
- 2-3 個題目
- 需要分析或跨領域理解

## 生成要求
1. 請生成 10 題 [DIFFICULTY] 難度的題目
2. 涵蓋不同文化類別
3. description 要有教育性，讓用戶學到知識
4. explanation 要清楚解釋為何答案正確
5. id 格式為 culture_001, culture_002... 依序編號

請直接輸出 JSON Array，不要加其他說明文字。
```

### 範例輸出

```json
[
  {
    "id": "culture_001",
    "category": "禮儀",
    "title": "古代見面禮節",
    "description": "在《紅樓夢》中，人物見面時的禮節體現了嚴格的等級秩序。晚輩見長輩要行大禮，如叩拜禮（跪地磕頭）；平輩相見則行揖禮或萬福禮。女子行萬福禮時，雙手交疊於小腹前，微微屈膝。林黛玉初見外祖母賈母時，依禮應行叩拜大禮，以示對至親長輩的尊敬。這種禮儀制度反映了中國傳統社會重視尊卑有序的倫理觀念。",
    "relatedChapters": [3, 4],
    "questions": [
      {
        "id": "q1",
        "question": "林黛玉初見外祖母賈母時，應該行什麼禮？",
        "type": "multiple_choice",
        "options": ["叩拜禮", "萬福禮", "揖禮", "鞠躬禮"],
        "correctAnswer": "叩拜禮",
        "explanation": "在古代，晚輩初次拜見長輩，特別是外祖母這樣的至親長輩，需要行叩拜大禮以示尊敬。"
      }
    ]
  }
]
```

## 4️⃣ 脂批解密 (commentary_decode) - CommentaryContent

### Prompt

```
你是一位紅樓夢專家，請為「脂批解密」任務生成脂硯齋批語解讀題目。

## 任務說明
用戶閱讀《紅樓夢》原文及脂硯齋的批語，分析批語的含義、批者的用意、以及與原文的關聯。

## 背景知識
脂硯齋是《紅樓夢》早期抄本的主要批註者，其批語被稱為「脂批」。批語形式包括：
- 甲戌本批語（甲戌側批、甲戌眉批、甲戌回前批等）
- 己卯本批語
- 庚辰本批語
- 畸笏叟批語

批語內容包括：
- 點評文字技巧
- 揭示隱藏伏筆
- 透露作者創作意圖
- 補充人物/情節背景
- 表達批者感慨

## 輸出格式 (JSON Array)
請生成 [DIFFICULTY] 難度的題目，每題格式如下：

```json
{
  "id": "commentary_XXX",
  "originalText": "《紅樓夢》原文",
  "commentaryText": "【批語來源：批語內容】",
  "chapter": 章回編號,
  "author": "批者名稱",
  "hint": "提示：幫助理解批語的背景或方向"
}
```

## 批者說明
- 脂硯齋：主要批註者，與曹雪芹關係密切
- 畸笏叟：另一位重要批註者
- 可標注具體版本如「甲戌側批」「庚辰眉批」等

## 難度要求

### Easy (簡單)
- 選擇含義較直白的批語
- 批語直接點明原文的意義或人物特點
- hint 提供較多引導
- 例如：點評人物初見的批語

### Medium (中等)
- 選擇需要一定背景知識的批語
- 批語涉及伏筆或隱喻
- hint 提供適度引導
- 例如：暗示人物命運的批語

### Hard (困難)
- 選擇含義深奧的批語
- 批語涉及全書結構、作者創作、或紅學爭議
- hint 提供較少引導，鼓勵獨立思考
- 例如：涉及八十回後情節的批語

## 生成要求
1. 請生成 10 題 [DIFFICULTY] 難度的題目
2. 選擇真實存在的脂批（可參考脂硯齋重評石頭記）
3. 涵蓋不同章回和批語類型
4. 批語要具有分析價值
5. id 格式為 commentary_001, commentary_002... 依序編號

請直接輸出 JSON Array，不要加其他說明文字。
```

### 範例輸出

```json
[
  {
    "id": "commentary_001",
    "originalText": "這個妹妹我曾見過的。",
    "commentaryText": "【甲戌側批：這是第一見，卻說曾見，可知前生有緣。】",
    "chapter": 3,
    "author": "脂硯齋",
    "hint": "這條批語點出了寶黛之間的前世因緣，呼應了第一回中神瑛侍者與絳珠仙草的故事。"
  },
  {
    "id": "commentary_002",
    "originalText": "滿紙荒唐言，一把辛酸淚。都云作者痴，誰解其中味？",
    "commentaryText": "【甲戌側批：此是第一首標題詩，能解者方有辛酸之淚，哭成此書。壬午除夕，書未成，芹為淚盡而逝。余嘗哭芹，淚亦待盡。每思覓青埂峰再問石兄，餘淚更有幾多，奈不遇癩頭和尚何！】",
    "chapter": 1,
    "author": "脂硯齋",
    "hint": "這條批語透露了曹雪芹創作此書的辛酸和脂硯齋與作者的深厚情誼，也暗示了作者的早逝和全書未完成的遺憾。「壬午除夕」指曹雪芹去世的時間。"
  }
]
```

## 🔄 快速生成流程

### Step 1: 選擇題目類型
複製對應的 Prompt

### Step 2: 替換難度
將 `[DIFFICULTY]` 替換為 `easy`、`medium` 或 `hard`

### Step 3: 生成題目
將 Prompt 貼到 AI（如 ChatGPT、Claude）生成題目

### Step 4: 驗證格式
確認輸出是有效的 JSON Array

### Step 5: 填入題庫
將生成的題目貼入 `question-bank.json` 對應的位置
